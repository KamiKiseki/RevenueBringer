autoyieldsystems.com must point at THIS Flask API only (server.py). If the browser shows a full old RevenueBringer marketing site (HTML/CSS) that is NOT this app, that host is still wired to a different Railway target.

Railway service name vs routing
  In /health, railway_service is whatever Railway sets (often "RevenueBringer").
  That value names THIS API deploy. It is NOT proof of wrong routing by itself.
  Wrong routing = you get a different app's HTML (e.g. legacy static deploy), not this site's layout + footer links to /health.

1) Open Railway → production. Open EVERY service → Settings → Networking / Custom domains.
2) Remove autoyieldsystems.com (and www if used) from every service EXCEPT the Flask API (the one that runs server.py at repo root; Railway may list it as RevenueBringer or another name—match the service you deploy from this repo).
3) On that service ONLY → add custom domain autoyieldsystems.com. Save.
4) Redeploy that service.
5) Test: curl https://autoyieldsystems.com/health
   Expect JSON: ok, service autoyieldsystems, marketing_site true, railway_service, railway_git_sha, railway_environment.
   Example fingerprint shape: railway_service=RevenueBringer, railway_git_sha=290ee73b5198, railway_environment=production.
6) Routing proof without HTML: curl https://autoyieldsystems.com/ops/deploy-proof
   Expect text/plain with FLASK API line plus Deploy fingerprint lines.
7) The public marketing site is GET / and GET /contact (HTML). Command Center (Reflex) is a separate process; it reads the same DATABASE_URL as Flask—including table contact_submissions from the contact form.
